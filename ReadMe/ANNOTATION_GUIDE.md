# 📝 人工标注模式 (Annotation Mode) 使用指南

## 🎯 标注模式是什么？

采样完成后，系统会自动加载录制的演示视频，你可以**逐帧手动浏览**，按键标注每一帧的识别结果是否正确。

**核心特性：**
- ✅ **逐帧手动浏览** - 按 'n'/'p' 手动控制前后帧（默认）
- ✅ **自动播放** - 按空格切换自动播放模式
- ✅ **快速标注** - 按 'a'/'r' 标注后自动跳下一帧

**用途：** 验证系统识别准确率，获得真实的性能评估数据。

---

## 🚀 启用标注模式

### 方法1：运行脚本自动启用（推荐）
```bash
bash run_camera_benchmark.sh
```

脚本中已默认包含 `--enable-annotation` 参数。

### 方法2：手动启用
```bash
python camera_interactive.py \
    --duration 30 \
    --enable-annotation \
    --db-path "benchmark\\YTF_100p.db" \
    --weights "weights\\model_best.pt"
```

## ⌨️ 标注按键说明

播放演示视频时，使用以下按键进行标注：

### 标注按键
| 按键 | 含义 | 操作 |
|------|------|------|
| **a** | Accept (接受) | ✅ 本帧识别**正确** → 自动跳下一帧 |
| **r** | Reject (拒绝) | ❌ 本帧识别**错误** → 自动跳下一帧 |

### 导航按键（手动模式）
| 按键 | 含义 | 操作 |
|------|------|------|
| **n** | Next | → 下一帧 |
| **p** | Previous | ← 上一帧 |

### 模式切换
| 按键 | 含义 | 操作 |
|------|------|------|
| **Space** (空格) | Toggle Mode | ⏸️/⏯️ 手动/自动播放切换 |

### 退出
| 按键 | 含义 | 操作 |
|------|------|------|
| **q** | Quit | 🚪 退出标注，完成评估 |

## 📺 两种标注模式详解

### 🟦 模式1：**手动逐帧模式**（默认）
```
【进入标注】
  ↓
【看第1帧】(静止显示)
  ↓
【决定是标注还是跳过】
  ├─ 按 'a'：标注为正确 → 自动跳下一帧
  ├─ 按 'r'：标注为错误 → 自动跳下一帧
  ├─ 按 'n'：跳到下一帧（不标注）
  └─ 按 'p'：回到上一帧（继续检查）
  ↓
【重复直到视频结束】
```

**优点：** 充分观看，不会遗漏，标注准确
**缺点：** 需要逐帧操作，比较慢

### 🟩 模式2：**自动播放模式**
```
【手动启用】
  按下 Space 键 → ⏯️ 自动播放
  ↓
【自动逐帧播放】(每 ~200ms 一帧)
  ↓
【在任意帧按 'a'/'r' 标注】
  标注后 → 继续自动播放，自动跳下一帧
  ↓
【到达末尾】
  自动停止，返回手动模式
```

**优点：** 观看连贯，像看视频
**缺点：** 可能反应不及，标注可能不够准确

```

【第2步】看着这个人脸，思考

【第3步】按 'a' 键
  ↓
  终端输出：✅ Frame 1: Accept
  ↓
【第4步】自动跳到下一帧并显示

  ↓

【看第9帧判断】
"第9帧是李四，第10帧是张三...

```

```
【按 Space 启用自动播放】

【到达最后一帧】

```

## 🎯 标注标准

| 判断标准 | 按键 | 说明 |
|--------|------|------|
| 认对了人 | **a** | 识别的人名与视频中人物一致 + 置信度 > 0.5 |
| 认错了 | **r** | 识别的人名与实际人物不符 |
| 没认出 | **r** | 识别结果为"Unknown"或置信度太低 |
| 看不清楚 | **p** 或 **n** | 先不标注，重新看或跳过，之后再补 |

## 💡 标注小贴士

### 1️⃣ **选择哪种模式？**
- 🟦 **手动模式**：首次使用、需要准确性 → 推荐
- 🟩 **自动模式**：已经熟悉、希望快速完成

### 2️⃣ **一次标注多少帧？**
- 推荐：**全部标注**（这样性能数据最准确）
- 最少：**至少标注 50%** 以上（样本足够统计）

### 3️⃣ **标注一个视频需要多长时间？**
- 30秒视频 ≈ 900帧
- **手动模式**：约 10-15 分钟（每帧1-2秒判断）
- **自动模式**：约 3-5 分钟（但需要快速反应）

### 4️⃣ **想重来怎么办？**
```bash
# 直接重新运行，会自动覆盖之前的标注
bash run_camera_benchmark.sh
```

### 5️⃣ **想只标注部分帧？**
可以，部分标注也有统计意义。比如标注前500帧，系统会计算这500帧的准确率。

## ❓ 常见问题

### Q: 标注太慢，怎么加速？
A: 使用自动播放模式
```
按 Space 切换到自动播放
可在代码中修改自动播放速度（default 200ms）
```

### Q: 某一帧看不清楚，怎么办？
A: 使用 'p' 回到那一帧多看几次
```
第10帧看不清 → 按 'p' 回到第9帧
重新看一遍第9、10帧 → 再做判断
```

### Q: 标注过程中按错了怎么办？
A: 无法撤销单个帧，但可以：
1. 重新运行标注（会覆盖之前的）
2. 或忽略，继续标注剩余的帧

### Q: 一个视频能标注多次吗？
A: 可以
```bash
# 第1次标注
bash run_camera_benchmark.sh

# 第2次标注
bash run_camera_benchmark.sh
(会覆盖)

# 想保存，改输出文件名
python camera_interactive.py ... \
    --output-report camera_eval/report_v2.json
```

### Q: 只想看不想标注？
A: 去掉 `--enable-annotation` 参数
```bash
python camera_interactive.py \
    --duration 30 \
    --db-path "benchmark\\YTF_100p.db" \
    --weights "weights\\model_best.pt"
    # 不加 --enable-annotation
```

### Q: 能自动标注吗？
A: 不能，标注必须人工判断。这是为了获得真实准确的性能数据。

## 💡 标注小贴士

### 1️⃣ **什么是"正确"？**
- ✅ 识别出了该人的身份（认对人）
- ✅ 置信度 > 0.5（识别有把握）

### 2️⃣ **什么是"错误"？**
- ❌ 识别出的人名不对（认错人）
- ❌ 识别出的人是"Unknown"（没认出）
- ❌ 置信度太低，不可信（< 0.5）

### 3️⃣ **如何高效标注？**
- 一遍过：第一次看视频就全部标注完
- 边看边标：识别结果立即标注，不要等
- 加速审查：如果很多帧都相同的人脸，可快速标注

### 4️⃣ **暂停的用途**
- 仔细观看不清晰的人脸
- 确认识别的人名和置信度
- 中途休息

## ❓ 常见问题

### Q: 标注太慢，能跳过吗？
A: 可以，改为不使用 `--enable-annotation` 参数：
```bash
python camera_interactive.py --duration 30
```

### Q: 标注过程中按错了怎么办？
A: 无法撤销，但可以：
1. 退出（按'q'）
2. 重新运行标注模式
3. 或者忽略这一帧，继续标注其他帧

### Q: 一个视频能标注多次吗？
A: 可以，重新运行：
```bash
python camera_interactive.py --enable-annotation \
    --output-report "camera_eval/report_v2.json"
```
会生成新的标注数据。

### Q: 标注结果保存在哪里？
A:
- CSV: `camera_eval/performance_metrics.csv`
- JSON: `camera_eval/camera_benchmark_report.json`

### Q: 看不清楚人脸，怎么办？
A: 按 **Space** 暂停，仔细观看后再继续。或者看原始MP4视频：
```bash
ffplay camera_eval/camera_demo_*.mp4
```

### Q: 能自动标注吗？
A: 不能，标注需要人工判断。这是为了获得真实准确的性能数据。
