# 实验数据集准备

## 3.1 数据集选择

本文采用**CASIA-WebFace**数据集进行开放集人脸识别实验。该数据集包含10,575个身份的人脸图像，具有丰富的姿态、表情、光照和年龄变化。为保证实验效率和可控性，我们选择其中**200个身份**进行实验，每个身份至少包含50张人脸图像。

### 3.1.1 特征提取

采用在MS-Celeb-1M数据集上预训练的**iResNet50**模型提取512维L2归一化特征向量。特征提取流程如下：

1. 使用MTCNN进行人脸检测
2. 对齐至标准尺寸(112×112像素)
3. 通过iResNet50提取特征
4. L2归一化处理

## 3.2 开放集实验协议

### 3.2.1 数据划分

为模拟真实开放集场景，将200个身份按**约45:55**比例随机划分为已知人和未知人：

| 类别 | 数量 | 用途 |
|------|------|------|
| 已知人(Known) | 89人 | 注册入库，系统识别对象 |
| 未知人(Unknown) | 100人 | 模拟陌生人，需被正确拒绝 |

**划分依据**：实际应用中，潜在陌生人数量通常超过注册用户，55%的未知人比例贴合实际场景。所有实验使用固定随机种子(seed=42)，确保结果可复现。

### 3.2.2 Gallery与测试集构建

对于每个已知人，进一步划分特征向量：

**Gallery集（注册库）**：
- 每人前10条特征向量
- 总计：89人 × 10 = **890条样本**
- 用途：构建识别参考库，计算自适应阈值统计量(μ, σ)

**Test Known集（已知人测试）**：
- 每人第11-20条特征向量
- 总计：89人 × 10 = **890条样本**
- 用途：评估已知类准确率(KCA)

**Test Unknown集（未知人测试）**：
- 100个未知人的所有特征向量
- 总计：**12,345条样本**（平均每人约123条）
- 用途：评估未知人检测率(UDR)

**设计考量**：
- Gallery每人10条样本：提供C(10,2)=45个genuine pairs用于估计μ和σ，确保统计量准确可靠
- Test每人10条样本：充分验证识别性能，降低随机误差影响
- Unknown全部样本：最大化测试覆盖度，模拟真实场景中陌生人的多样性

## 3.3 数据集统计

准备完成后的实验数据集包含：


## 3.4 评估指标

### 3.4.1 核心指标

**已知类准确率(Known Class Accuracy, KCA)**：
$$
\text{KCA} = \frac{\text{正确识别的已知人数}}{\text{已知人测试样本总数}} \times 100\%
$$

**未知人检测率(Unknown Detection Rate, UDR)**：
$$
\text{UDR} = \frac{\text{正确拒绝的未知人数}}{\text{未知人测试样本总数}} \times 100\%
$$

**开放集识别率(Open-Set Recognition Rate, OSR)**：
$$
\text{OSR} = \frac{\text{正确识别的已知人} + \text{正确拒绝的未知人}}{\text{测试样本总数}} \times 100\%
$$

### 3.4.2 辅助指标

- **精确率(Precision)**：预测为未知人中真正未知的比例
- **F1分数(F1-Score)**：精确率与召回率的调和平均数
- **平均延迟(Average Latency)**：单次查询的平均推理时间(毫秒)

## 3.5 阈值分析与设定

### 3.5.1 相似度校准

为增强高相似度区间的区分能力，采用分段线性校准函数对原始余弦相似度进行变换：

$$
\text{score}_{\text{cal}} = \text{similarity\_con}(\text{score}_{\text{cos}})
$$

该校准函数在(0.6, 1.0)区间放大分数差异，有效区分真实匹配与高质量攻击样本。

### 3.5.2 最优阈值选择

基于Gallery集计算genuine对(1,028对)和impostor对(100,000对)的校准相似度分布：

- **等错误率(EER)点**：τ = 0.3073 (FAR = FRR = 1.75%)
- **实际部署阈值**：τ = 0.45 (FAR ≈ 5%, FRR ≈ 2%)

**阈值选择策略**：相比EER点，采用更保守的τ=0.45可有效降低误接受率(FAR)，在安全敏感应用中更为适用。

**自适应阈值设定**：
- 下界：0.4（确保所有个性化阈值合理）
- 上界：由μ - 2σ公式自动确定

## 3.6 可重复性保证

所有数据集准备脚本已开源，关键步骤如下：

```bash
# 1. 提取CASIA特征 (200个身份)
python scripts/extract_casia_features.py --num-ids 200

# 2. 准备开放集数据 (89 known / 100 unknown)
#    每个known person: 10条gallery + 10条test
python scripts/prepare_open_set_optimized.py \
    --db casia \
    --num-known 89 \
    --gallery-size 10 \
    --test-size 10

# 3. 执行评估
python scripts/compare_with_prepared_data.py \
    --data-dir benchmark/open_set_data_casia200
```

所有实验使用固定随机种子(seed=42)，确保结果可复现。

**说明**：200个身份中有11个因样本数不足20条（gallery_size + test_size）被过滤，最终有效身份为189个。
